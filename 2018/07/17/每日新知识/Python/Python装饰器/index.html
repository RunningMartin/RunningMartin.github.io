装饰器的实现

装饰器的最大作用是能在任意时间添加额外的处理过程。一般用于模式设计，改善代码结构。

  • 原理

  • 实现

  • 类实现的装饰器，类实现的装饰器应用于实例方法时，会导致方法绑定丢失：这是因为类实现的装饰器没有实现描述符协议，因此在装饰器实例替换方法时，导致实现绑定的描述符方法被隐藏，无法自动调用。但是函数对象默认实现了描述符协议和绑定规则。

  • 嵌套：多个装饰器时，遵循从上往下的原则，这里需要注意，如果在对类型方法使用装饰器时，我们应该让类型方法的装饰器放最外面，这样才不会导致绑定方法失效。

  • 参数：装饰器也可以接受参数。

  • 属性：可以通过functools.wraps复制原对象的属性。

  • 类型装饰器：作用于类型的装饰器

应用

  • 调用追踪

  • 属性管理：为目标添加额外属性，在原有设计上以装配方式混入其他功能。

  • 实例管理：替换目标构造方法，拦截用例创建。用于实现对象缓存或单例模式

  • 部件注册：用装饰器代替配置文件实现路由注册。
