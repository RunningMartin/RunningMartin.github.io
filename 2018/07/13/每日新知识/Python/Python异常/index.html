每日新知识--Python异常

Posted on 2018-07-13 Edited on 2019-10-26 In 每日新知识 , Python基础 , 《Python3 学习笔记》阅读笔记

记录关于Python异常处理的相关知识点。

在面对错误时，常见的错误处理方式为：

  • 返回错误码，由调用方决定如何处理错误，这种方式不需要额外状态保存和流程控制机制非常简单、高效(采用此方法，需要将所有可能的错误码放入函数签名中，便于调用者了解)。
  • 采用专门的语句处理，这样可以分离正常的逻辑和故障处理代码。

执行机制

  • 解释器会使用块栈的结构来处理异常逻辑。在执行期，相关指令会提前将跳转位置存储到块栈中，需要时直接获取。

  • 异常会从触发地向外传递，直到进程奔溃或者被捕获。

  • 触发异常后，解释器会逐级为每个未能捕获的异常的函数调用创建关联追踪对象，它负责存储栈帧、最后指令的位置、以及源代码的信息。这些对象通过链表来串联起来。

异常处理

  • 语法

  • 在捕获异常时，最好明确异常的类型，如果需要同时匹配多个异常，可以使用元组进行包装。

  • 抛出异常：raise 异常(若在except中，没有异常，则视做为重新抛出异常)。

自定义类型

  • 继承BaseException或Exception即可。异常信息会存在Exception.args中，但是在创建异常时，最好还是显式使用属性来存储。

  • 异常链：通常解释器会将前一个异常对象存储在__context__中，但是我们可以通过``from语法，在保存原异常的前提下，主动构造和选择目标，触发新的异常。因此如果我们从from None则可以打破链条。如果异常未被捕获，解释器会自动输出cause和context`中的异常信息。

模式

  • 我们可以通过在不同的层次设置不同的异常，在内部通过触发异常的方式，进行跳转。可以终止多级函数调用过程。

警告信息

  • 可以通过标准模块warning输出警告信息。
