文章摘要

开放类

Python可以在运行期动态向实例或类型添加成员甚至是方法。

  • 成员：通过类型或者实例直接进行赋值即可加入到相应的名字空间中。

  • 方法：方法只能添加到类的名字空间中。

  • 但是对于祖先根类object来说，是无法向其类型或实例添加新的成员的，因为其根本没有__dict__。但是可以通过SimpleNamespace(对object的简单继承)来直接创建实例。

__slots__

  • 名字空间字典虽然带来了很多便利，但是对于需要创建海量实例的类型来说，这会造成很严重的内存和性能问题，因此可以通过__slots__来进行固话处理。解释器会阻止实例创建__dict__等成员，仅仅为指定成员分配空间，如果添加任何未预置的属性则会引发异常。其原理是：解释器在创建类型对象时，会直接将指定成员包装成描述符静态分配到类型对象的尾部。实例字段不再存储到__dict__中，而是直接分配引用的内存。因此只能修改引用的内容，而无法改变成员的名字。

  • __slots只能用于约束实例，而且不能将__dict__放入其中。

  • 如继承的父类中设置了__slots__，子类不设置的话，则实例会创建__dict__。若设置了，则向父类__slots__的追加。
