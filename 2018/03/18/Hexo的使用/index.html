Hexo的使用

Posted on 2018-03-18 Edited on 2019-10-26 In Hexo

记录了一些在hexo刚开始时的一些常用配置。

Hexo的使用

依赖安装

详细参考

  • 安装npm： sudo apt install npm

  • npm换源： sudo npm install -g cnpm --registry=https://registry.npm.taobao.org

  • 安装git：sudo apt install git

  • 安装Hexo：sudo cnpm install -g hexo-cli

  • 验证：

创建存储仓库

  • 创建的存储仓库可以用github或者国内的coding，仓库名为：
    • github：username.github.io
    • coding：username.coding.me
  • coding创建好项目后，可以设置pages，详情，请查看coding。

博客使用

  • 创建：hexo init username.github.io

  • 安装主题：

  • 配置

    • 站点配置：站点配置的文件在博客根目录下的_config.yml中，具体参考。
    • 主题配置，主题配置的文件在themes中主题文件夹下的_config.yml中，next主题的具体配置参考。
      • 在这里重点强调一点，设置disqus评论时，shortname: '你在disqus中设置'

  • 配置数学公式支持

    • 在next的_config.yml将mathjax设置为true。
  • 公式格式
    • 行内公式：$\sin ^{ 2 }{ \theta +\cos ^{ 2 }{ \theta =1 } }$
    • 效果：$\sin ^{ 2 }{ \theta +\cos ^{ 2 }{ \theta =1 } }$
    • 行公式：$$\frac { dy }{ dx } =\frac { { e }^{ x } }{ 3{ y }^{ 2 } }$$
    • 效果：$$\frac { dy }{ dx } =\frac { { e }^{ x } }{ 3{ y }^{ 2 } }$$
    • 如果有冲突可以使用\转义。

  • 文章摘要配置

    • 推荐：在文章中使用\进行手动截断。
    • 自动形成摘要：
      • 在主题配置中，将auto_excerpt改为true。

  • 测试：在博客根目录下执行hexo server，访问：http://localhost:4000即可看到

  • 创建文章hexo new [layout] <title>具体参考

  • 发布

配置百度谷歌搜索引擎验证

准备工作

  • 百度搜索引擎验证
  • 谷歌搜索引擎验证
  • 进入上面两个网站，选择推荐的方法，进行HTML文件验证，只需要将这相应的文件下载，放在public中，然后推送到网站中，点击验证即可。

站点设置

  • 生成站点地图

  • 设置站点配置文件_config.yml

  • 编译推送即可。

收录

  • Google站长工具 提交sitemap即可。

  • 百度收录：搜索资源平台——>数据引入——>链接提交

参考文章

  • Hexo插件之百度主动提交链接
