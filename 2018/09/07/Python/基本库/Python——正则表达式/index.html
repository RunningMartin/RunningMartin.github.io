字符匹配

基本元字符

元字符 说明
. 匹配除换行符以外的任意单个字符
| 逻辑或操作(从左到右依次匹配)
[] 匹配字符集合中的一个字符
[^] 对字符集合求非
- 定义一个区间(如[a-z]，区间内的值可以有多个)
\ 对下一个字符转义

数量匹配

元字符 说明
* 前一个字符(子表达式)重复0次或多次
*? *懒惰版本
+ 前一个字符(子表达式)重复1次或多次
+? +懒惰版本
? 前一个字符(子表达式)重复0次或1次
{n} 前一个字符(子表达式)重复n次
{m,n]} 前一个字符(子表达式)重复至少m次，最多n次
{n,} 前一个字符(子表达式)重复至少n次
{n,}? {n,}懒惰版本

位置匹配

元字符 说明
^ 匹配字符串开头
\A 匹配字符串开头
& 匹配字符串结尾
\Z 匹配字符串结尾
\b 匹配单词的边界，位于\w和\W之间。
\B \b的反义
(?m) 分行匹配，放最前面，将改变^和$的行为，能对每行进行匹配。

子表达式

  • 子表达式：将正则表达式用括号括起来。支持嵌套操作。

    • 便于理解：(((\d{1,2})|(1\d{2})|(2[0-4]\d)|(25[0-5]))\.){3}((\d{1,2})|(1\d{2})|(2[0-4]\d)|(25[0-5]))：匹配IPv4地址。
    • 回溯引用：能引用前面的匹配结果，从而达到符合自己要求的匹配。
      • [ ]+(\w+)[ ]+\1：匹配重复的两个单词，\1表示引用第一个子表达式的结果，若(\w)+：匹配到of，\1也代表of。
    • 替换：$1表示此处的内容为第一个子表达式所匹配到的内容。
    • 替换时进行大小写转换
    元字符 说明
    \E 结束\L或\U转换
    \l 把下一个字符转换为小写
    \L 将\L到\E之间的字符转换为小写
    \u 将下一个字符转换为大写
    \U 将\U到\E之间的字符转换为大写

前后查找

  • 前后查找只提供定位，不返回匹配的内容。

  • 向前查找：(?=模式)，在符合的模式之前进行查找。\w+(?=:)：可以用于查找URL地址中的协议名。内容中可以包含.和+，其长度可变。

  • 负向前查找：(?!模式)，在与给定模式不匹配的位置之前查找。

  • 向后查找：(?<=模式)，在符合的模式之后进行查找。其长度只能是固定的。

  • 负向后查找：(?<!模式)，在与给定模式不匹配的位置之后查找。

嵌入条件

  • 根据回溯引用进行条件处理：
    • 格式：(?(子表达式)成立时的正则表达式[|不成立时的正则表达式])：不成立时的正则表达式可选。
    • 示例：(<[Aa]\s[^>]+>\s*)?<[Ii][Mm][Gg]\s+[^>]+>(?(1)\s*</[Aa]>)：匹配带属性的或标签的img标签，或者是没有或标签的img标签。
  • 根据前后查找进行条件处理
    • 格式：(?(前后查找条件)成立时的正则表达式)
    • 示例\d{3}(?(?=[- ])[- ]\d{8})：匹配座机号。

写作正则表达式的思路

  • 将所有的可能性都考虑到比将所有的不可能的情况排除掉更容易。

re模块

初步使用

  • re.match(pattern, string, flags=0)：从string的开头进行匹配，匹配成功，返回match对象，如果开头匹配不成功，则返回None。

  • re.search(pattern, string, flags=0)：和match相同，只是能在任意位置匹配。

  • re.findall(pattern, string, flags=0)：用列表列出所有匹配的结果。

  • re.fullmatch(pattern, string, flags=0)：若string与pattern完全匹配，则返回匹配对象。

  • re.split(pattern, string, maxsplit=0, flags=0)：按pattern将string进行拆分，maxsplit指定最大拆分次数(为0则没有限制)。 如果有捕获符号，则匹配的内容也将返回。

  • re.finditer(pattern, string, flags=0)：返回一个match对象的迭代器对象

  • re.sub(pattern, repl, string, count=0, flags=0)：string匹配到的内容替换到repl中的相关内容。

复用

  • re.compile(pattern, flags=0)：生成正则表达式对象，可以复用。

  • 其他方法和re相同。
