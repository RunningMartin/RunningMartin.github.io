多进程

  • 多进程：假设CPU为一个工厂，CPU的核心数为工厂的车间数。如果我们的CPU为单核，CPU一次就只能处理一个任务。但是如果我们的CPU是多核的，CPU就能同时处理多个任务。

    multiprocessing

  • 介绍：在Python中，multiprocessing模块为我们提供了在多平台使用多进程的能力。而且我们使用子进程能够有效的避免Python中多线程中GIL的坑，这个坑的详情请见Python 多线程。

    Process类

  • 简介：Process类提供与进程相关的API。

  • 创建进程

  • 设置启动方法

    方法 说明
    spawn 父进程会创建一个全新的子进程，子进程只会继承运行run()方法所必须的资源。但是这个方法比fork和forkserver慢很多。这个方法在Unix和Windows中都支持，为在Windows中为默认的方法。
    fork 父进程采用os.fork()来创建子进程。其和父进程是相同的，子进程会继承父进程所有的资源，但是在fork一个多线程的进程时存在安全性问题。只在Unix中有用。
    forksever 当父进程启动时，会启动一个server进程，当需要创建一个新的进程时，父进程会和server进程通信，并通过fork创建一个新的进程。只继承必要的资源，只运行在支持通道的Unix操作系统中。

  • 范例

  • spawn或forkserver作为启动方法时的注意事项：

    • Process初始化的参数都必须能够pickle。
    • 子进程访问全局变量时，值可能与父进程中的值不同。
    • 为了确保主modul能被新的解释器引入，程序的入口应该放在__name__ == '__main__'中。

  • 注意事项：

    • 避免共享状态：尽可能的避免共享状态，如果需要共享，请使用pipe或queue。
    • 确保代理方法的参数是可以进行pickle。
    • 代理的线程安全：不要在多线程中使用代理对象，除非这个代理对象有锁。在多进程中使用则是安全的。
    • join僵尸进程：僵尸进程：当一个进程完成了，但是还没有join。僵尸进程不会太多，因为当新的进程开始、active_children或Process.is_alive被调用时，所有没有join的已经完成的进程将join，但是显示调用join方法也是有必要的。
    • 避免调用进程的terminate方法：调用terminate方法会让被该进程使用的共享数据被破坏或不能被其他应用使用。最好在没有共享数据的进程中使用。
    • 进程使用queue传递数据时，应当小心使用join方法：因为queue是通过pipe实现的，因此存入数据的进程会等待所有缓存的数据都加入pipe后才能被关闭，这也就意味着只有当传入的数据都被remove后才能join。
    • 显式传递资源给子进程：将资源作为子进程的参数传递，这样可以避免父进程的垃圾回收器将其回收。
    • 避免使用类文件对象替换流输入：因为类文件输入会存在缓存问题，如果一定要使用，最好去检查下pid，如果pid变换了，则抛弃掉缓存。

Manager

Manager提供一个在不同进程之间共享内存的方法，即便这些进程是分布式的，也可以进行共享。它将启动一个服务进程来管理共享数据，其他的进程能通过使用代理来访问这些共享数据。当它被垃圾管理器回收或者其父进程关闭时，管理进程将会关闭。

  • 说明
方法名 说明
BaseManager([address[, authkey]]) address为元组：(ip,port)，authkey为用来区分验证的字符串。
start() 启动一个子程序来启动manager。
get_server() 返回Manager实际控制的服务，调用serve_forever()可以启动这个服务。
connect() 链接一个远程manager进程。
shutdown() 关闭manager使用的进程。
register() 重点+核心，详情请看下面的范例。

Proxy Object
  • 说明：
    • 用于引用一些共享的对象，这些对象可能会存在不同的进程中。
    • 能够嵌套使用。
    • 需要主要一点是，如果传入的是标准的dict对象或list对象，通过外部修改是不能进行直接修改的，必须更新。通过代理对象可以直接修改，详情请看范例。

Pool

  • 说明：提供一个进程池。
  • 初始化参数说明
参数 说明
processes 进程池中的进程个数，默认为cpu个数。
initializer 初始化进程池时调用
initargs initializer的参数
maxtasksperchild 每个进程能完成的最大任务数，当任务数达到，则重新创建。
context 指定上下文
  • 示例
  • 方法说明
方法 说明
apply(func[, args[, kwds]]) 采用阻塞的方式运行
apply_async(func[, args[, kwds[, callback[, error_callback]]]]) 异步的形式执行
map(func, iterable[, chunksize]) 和内建的map相同，将iterable切割为多块，将每块作为单独的任务提交给进程池，chunksize指定每块的大小
map_async(func, iterable[, chunksize[, callback[, error_callback]]]) 异步
imap(func, iterable[, chunksize]) 和map相同，只是返回的结果是一个生成器。
imap_unordered(func, iterable[, chunksize]) 无序
starmap(func, iterable[, chunksize]) 当iterable的元素是也是可迭代的，则会将元素直接解包后传递给func
starmap_async(func, iterable[, chunksize[, callback[, error_callback]]]) 异步
close() 关闭进程池，但是会等待有任务的进程完成任务
terminate() 立即关闭进程池
join() 让所有的进程退出，必须先调用closer或者terminate
