表操作

  • 创建表

    • 语法

    • 细节：

      • STATIC定义：设置STATIC，则具有相同的partition key的数据这个字段的值都相同。

        • 限制：
          • 定义表时使用了COMPACT STORAGE，不能使用static。
          • 没有clustering columns的表是不能使用static的，因为每个partition只有一行，所有本质上就是static的。
          • 只有不在PRIMARY KEY中的列才能是静态的。
        • 示例：

      • Primary Key

        • 含义；在表中，区别不同行的数据是通过primary key来区别的，primary key包含两个部分partition key和clustering columns。

        • partition key:primary key的第一个参数为partition key。可以为一个列表

        • clustering columns：primary key除了第一个参数，都为clustering columns。

        • 示例

      • 表的选项

        • COMPACT STORAGE:压缩存储(在创建表后，不能被修改)。这个字段存在是因为为了兼容CQL3之前，在新的表中不推荐使用，他会带来以下的限制：

          • 不能使用collections和static columns。
          • 如果至少有一个clustering column，那么它必须在primary key之外有一列。这意味在创建后，某些特殊情况下无法添加或删除列。
          • 在创建索引上是受限的，并且不能创建materialized view。

        • CLUSTERING ORDER:设置排序(在创建表后，不能被修改)。只能设置clustering columns，默认采用ASC

        • 其他选项参见官方文档
